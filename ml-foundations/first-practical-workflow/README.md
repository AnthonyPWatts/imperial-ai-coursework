# A First Practical Machine Learning Workflow

This folder contains a 12-part incremental code-along for machine-learning prerequisite prep.

The aim is not to be a complete machine-learning textbook. It is a practical bridge from the core ideas - features, rows, models, predictions, loss, overfitting, scaling, pipelines, and evaluation - to the kind of workflow used in scikit-learn notebooks.

Each document is deliberately small and self-contained. The rule throughout is: **nothing appears before it is explained**. Imports, objects, model classes, metrics, and array shapes are introduced only when they are needed.

## Suggested order

1. Setup and notebook contract
2. Start with a tiny dataset you can see
3. Split before training
4. Fit a regression model
5. Measure wrongness: error, residual, loss
6. Repeat with a built-in dataset
7. Deliberately overfit something
8. Classification: predicting a category
9. Accuracy can flatter a stupid model
10. Scaling: stop big-number features taking over
11. Pipeline: make the safe path the easy path
12. Cross-validation and final recap

## How to use these files

Use the PDFs for reading and the DOCX files if you want to edit the wording or paste sections into your own course-prep notes. Work through the exercises in order, creating one notebook of your own as you go.

The examples are intentionally small. They are meant to make the workflow visible, not to produce impressive model performance.
